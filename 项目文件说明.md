# 项目文件说明

## 1. 项目概述

本项目为“西安地铁换乘路径计算器”。系统基于真实西安地铁线路数据，使用 C 语言实现核心图结构、最小堆与 Dijkstra 路径搜索算法，使用 Python + Flask 提供后端服务，并通过 HTML / CSS / JavaScript + Leaflet 构建网页可视化界面。

当前系统支持的主要功能包括：

- 时间最短路径查询
- 换乘最少路径查询
- 综合最优路径查询
- 备用方案生成
- 时段调度权重调整
- Dijkstra 算法过程可视化
- 本地结果与高德地铁路径结果对比分析

当前运行数据规模如下：

- 线路数：13
- 物理站点数：250
- 换乘站数：37
- 状态节点数：289
- 有向边数：636

## 3. 运行方式

### 3.1 一键启动主项目

在项目根目录执行：

```powershell
.\start_project.bat
```

或：

```powershell
powershell -ExecutionPolicy Bypass -File .\start_project.ps1
```

启动成功后，在浏览器访问：

```text
http://127.0.0.1:5000
```

### 3.2 手动启动主项目

在项目根目录执行：

```powershell
python metro_router/app.py
```

手动运行所需环境：

- Python 3.10 及以上
- Flask
- 可用的 `metro_router/core/metro_router.exe`

如果修改了 C 源码但尚未重编译，可先执行：

```powershell
cd metro_router/core
mingw32-make
```

### 3.3 重建运行数据

如果需要从原始 Shapefile 重新生成运行数据，可执行：

```powershell
python metro_router/data_loader.py
```

数据重建依赖：

- Python 3.10 及以上
- `geopandas`
- `pandas`

执行后会重新生成：

- `metro_router/data/graph.txt`
- `metro_router/data/stations.json`
- `metro_router/data/routes.json`

### 3.4 运行对比测试

对比测试系统的说明见 `compare/README.md`。

可直接运行：

```powershell
python -m compare.run_compare
```

也可以使用已提供的启动脚本：

```powershell
.\compare\start_compare.bat
```

或：

```powershell
powershell -ExecutionPolicy Bypass -File .\compare\start_compare.ps1
```

## 4. 主程序目录说明

### `metro_router/`

这是主项目代码目录，负责地铁路径计算与网页展示，主要包含以下几部分：

- `app.py`
  - Flask 后端入口，负责加载运行数据、调用 C 核心程序、提供 API，并对路径结果做二次整理

- `data_loader.py`
  - 读取西安地铁原始 Shapefile，生成 `graph.txt`、`stations.json`、`routes.json`

- `data/`
  - 主项目运行时直接使用的数据文件

- `templates/index.html`
  - 页面 HTML 模板入口

- `static/`
  - 前端样式与脚本，包括主查询逻辑、路径分段逻辑、调度面板和 Dijkstra 可视化

- `core/`
  - C 语言核心算法部分，包括图结构、最小堆、Dijkstra 实现、命令行入口和测试程序

### `metro_router/core/` 的主要文件

- `graph.h` / `graph.c`
  - 图结构与图加载逻辑

- `min_heap.h` / `min_heap.c`
  - Dijkstra 使用的最小堆实现

- `dijkstra.h` / `dijkstra.c`
  - 核心路径搜索算法实现

- `main.c`
  - C 程序入口，负责读取起终点并输出 JSON 结果

- `test_core.c`
  - C 层测试代码

- `metro_router.exe`
  - 当前仓库中可直接调用的 C 核心可执行文件

## 5. 对比测试目录说明

### `compare/`

该目录用于执行“本地路径规划结果”与“高德地铁路径结果”的对比测试，不参与主系统网页运行，但可用于结果校验和实验分析。

主要文件如下：

- `config.py`
  - 对比测试配置，包括高德 Key、城市参数、超时、重试与样例数量

- `local_router.py`
  - 对主项目本地查询逻辑的轻量封装

- `amap_client.py`
  - 调用高德接口并解析返回结果

- `test_case_generator.py`
  - 生成手工样例与随机样例

- `comparator.py`
  - 比较本地结果与高德结果

- `reporter.py`
  - 输出 JSON / CSV 结果文件

- `generate_report.py`
  - 根据结果 JSON 生成 Markdown 汇总报告

- `run_compare.py`
  - 对比测试主入口

- `start_compare.bat` / `start_compare.ps1`
  - 对比测试的一键启动与环境检查脚本

- `output/`
  - 对比测试输出目录，保存 `results_*.json`、`summary_*.csv`、`stats_*.json` 等结果文件

运行 `compare/` 时需要注意：

- 依赖网络访问高德接口
- 依赖可用的高德 API Key
- 对比测试耗时明显长于普通本地查询

## 6. 数据与来源目录说明

### `CPTOND-2025/`

该目录存放城市公共交通数据处理代码及整理后的数据结果，是主项目所用原始数据来源之一。

其中：

- `CPTOND-2025/code/`
  - 数据采集、清洗、Shapefile 生成、区间拆分和按城市整理脚本

- `CPTOND-2025/city_list/`
  - 城市清单及辅助表

- `CPTOND-2025/dataset/`
  - 已生成的公共交通数据结果

当前主项目主要使用的是：

- `CPTOND-2025/dataset/metro/shapefiles/sian/`

中的西安轨道交通数据，再由 `metro_router/data_loader.py` 转换为主程序运行所需格式。
