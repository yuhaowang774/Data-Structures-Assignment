# 协作开发指南

## 项目概述

西安地铁换乘最优路径计算器，包含三个独立分支，三人各负责一个分支。

## 分支分配

| 分支 | 职责 | 技术栈 |
|------|------|--------|
| `master` | 完整前后端一体版 | Flask + C (Dijkstra) + Leaflet.js |
| `pure-frontend` | 纯前端静态版 | HTML/JS + Leaflet + 内置 JSON，JS 重写 Dijkstra |
| `flutter-app` | Flutter 移动端 | Flutter/Dart + flutter_map + Dart Dijkstra |

## Git 工作流

### 初始设置

仓库地址：`https://github.com/yuhaowang774/Data-Structures-Assignment`

```bash
git clone https://github.com/yuhaowang774/Data-Structures-Assignment.git
cd Data-Structures-Assignment

# 每人切换到自己的分支
git checkout master          # 成员 A
git checkout pure-frontend   # 成员 B
git checkout flutter-app     # 成员 C
```

### 日常开发流程

```bash
# 1. 开始工作前，拉取最新代码
git pull origin <你的分支>

# 2. 开发完成后提交
git add .
git commit -m "feat: 简要描述改动内容"

# 3. 推送到远程
git push origin <你的分支>
```

### 提交信息规范

```
feat: 新功能
fix: 修复 bug
docs: 文档变更
refactor: 重构（不改变功能）
style: 格式调整（不影响逻辑）
```

### 禁止事项

- **不要**直接推送或修改其他人的分支
- **不要**在非自己分支上 commit
- **不要**force push（`git push -f`）
- 如需引用其他分支的代码，手动复制，不要 merge

## 共享资源

以下资源三个分支共用，修改前需通知全组：

| 文件 | 说明 |
|------|------|
| `doc/` | 设计文档目录 |
| `amap_line8_line15.json` | 高德地图 8/15 号线数据 |
| `amap_stations_result.json` | 高德站点数据 |
| `fetch_osm_data.py` | OSM 数据抓取脚本 |
| `generate_line8_line15.py` | 数据生成脚本 |

修改共享资源时：
1. 在群里通知所有人
2. 等确认后再 push

## 数据文件同步

三个分支使用相同格式的 `graph.json` 和 `routes.json`，数据结构必须保持一致：

- `graph.json`：邻接表图结构，节点为站点 ID
- `routes.json`：线路信息，包含站点坐标和线路颜色

如果有人修改了数据格式，必须通知其他两人同步更新。

## 冲突处理

由于三人各守一个分支，正常情况下不会产生合并冲突。但需注意：

1. **共享文件冲突**：如果两人同时修改 `doc/` 下的文件，后 push 的人需要 `git pull` 后解决冲突
2. **数据格式变更**：修改 JSON 结构前必须全组讨论

## GitHub 协作设置

### 1. 添加协作者

仓库所有者操作：

1. 打开 `https://github.com/yuhaowang774/Data-Structures-Assignment/settings/access`
2. 点击 **Add people**，输入另外两人的 GitHub 用户名或邮箱，发送邀请
3. 对方在邮箱或 GitHub 通知中点击 **Accept invitation**

默认权限为 **Write**，可推送到任意分支。

### 2. 设置分支保护（推荐）

防止误推到别人的分支：

1. 进入 `https://github.com/yuhaowang774/Data-Structures-Assignment/settings/branches`
2. 对每个分支点击 **Add branch protection rule**：

| Branch name pattern | 设置 |
|---|---|
| `master` | ✅ Require a pull request before merging，Restrict who can push → 仅成员 A |
| `pure-frontend` | ✅ Require a pull request before merging，Restrict who can push → 仅成员 B |
| `flutter-app` | ✅ Require a pull request before merging，Restrict who can push → 仅成员 C |

具体操作：勾选 **Require a pull request before merging** → 勾选 **Restrict who can push to matching branches** → 添加对应负责人。

### 3. 配置本地 Git 用户信息

每人克隆后执行（替换为自己的信息）：

```bash
git config user.name "你的名字"
git config user.email "你的邮箱"
```

### 4. Personal Access Token

推送时 GitHub 不接受密码，需使用 Token：

1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. **Generate new token (classic)**
3. 勾选 `repo` 权限，生成并复制 Token
4. 推送时密码处粘贴 Token

### 5. 验证连通性

每人执行一次测试推送：

```bash
echo "test" > test.txt
git add test.txt
git commit -m "test: 连通性测试"
git push origin <你的分支>
# 验证成功后删除
git rm test.txt
git commit -m "chore: 删除测试文件"
git push origin <你的分支>
```
