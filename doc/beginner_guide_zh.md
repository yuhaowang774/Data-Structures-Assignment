# 初学者阅读指南

## 1. 项目核心问题

本项目要解决的问题是：在真实西安地铁网络中，根据起点站和终点站，计算满足不同优化目标的乘车方案，并以可视化形式展示结果。

从数据结构与算法的角度看，这个问题可以概括为：

- 如何把地铁网络抽象为图
- 如何表示换乘
- 如何在图上高效求最优路径
- 如何把算法结果恢复为可读的乘车步骤

## 2. 推荐阅读顺序

建议按照以下顺序阅读代码：

1. `metro_router/core/graph.h`
2. `metro_router/core/min_heap.h`
3. `metro_router/core/dijkstra.h`
4. `metro_router/core/min_heap.c`
5. `metro_router/core/graph.c`
6. `metro_router/core/dijkstra.c`
7. `metro_router/core/main.c`
8. `metro_router/app.py`
9. `metro_router/static/script.js`

这个顺序有助于先理解数据结构和算法，再理解后端与前端如何围绕算法工作。

## 3. 图模型的理解

本项目没有采用“一个物理站点对应一个图节点”的简单建模方式，而是采用：

- `站名 + 线路` 作为状态节点

例如：

- `小寨-地铁2号线`
- `小寨-地铁3号线`

它们是两个不同的节点。

这种建模方式的作用在于：

- 显式表示换乘行为
- 精确统计换乘次数
- 在路径恢复时清楚说明“在哪一站、从哪条线换到哪条线”

## 4. 图中边的含义

图中的边分为两类：

### 行车边

表示同一条线路上相邻站点之间的连接。

边权表示：

- 行车时间

### 换乘边

表示同名站点、不同线路状态节点之间的连接。

边权表示：

- 固定换乘时间（当前为 2 分钟）

## 5. 为什么使用 Dijkstra 而不是 BFS

本项目不能直接使用 BFS，主要原因是：

- 图中边权不相等
- 行车边和换乘边都带有时间代价

BFS 适用于无权图或等权图，只能保证边数意义下的最短路径，不能保证总时间最短。

因此，本项目使用扩展 Dijkstra，以支持：

- 时间最短
- 换乘最少
- 综合最优

## 6. 最小堆的作用

Dijkstra 每一轮都需要从候选节点中取出当前代价最小的节点。为了避免每次都线性扫描全部未访问节点，本项目实现了手写最小堆。

最小堆位于：

- `metro_router/core/min_heap.h`
- `metro_router/core/min_heap.c`

复杂度如下：

- `heap_insert`: `O(log n)`
- `heap_extract_min`: `O(log n)`
- `heap_is_empty`: `O(1)`

## 7. 三种路径模式的理解

### `mode=0`

目标为：

- 时间最短

### `mode=1`

目标为：

- 换乘最少
- 当换乘次数相同时再比较总时间

当前实现通过代价函数：

- `transfers + total_time * 1e-6`

来保证换乘次数是主优先级，时间是次优先级。

### `mode=2`

目标为：

- 综合最优

当前源码已经支持该模式。

## 8. Python 与前端在系统中的角色

### Python 的角色

Python 在本项目中承担两类任务：

- 数据预处理
- 后端桥接

对应文件：

- `metro_router/data_loader.py`
- `metro_router/app.py`

### 前端的角色

前端负责：

- 地图展示
- 站点搜索
- 查询触发
- 路径结果展示
- 备用方案切换
- 调度面板交互
- 算法可视化

前端并不替代 C 核心算法，而是围绕后端结果完成组织和展示。

## 9. 当前数据规模

以当前运行数据为准：

- 13 条线路
- 250 个物理站点
- 37 个换乘站
- 289 个状态节点
- 636 条有向边

理解这些规模信息有助于认识：

- 图是否属于稀疏图
- 为什么采用邻接表
- 为什么最小堆优化有意义

## 10. 容易混淆的几个问题

### 为什么站点数和节点数不相同

因为换乘站会被拆分成多个状态节点。

### 为什么会同时存在 `graph.txt`、`stations.json` 和 `routes.json`

因为：

- `graph.txt` 供 C 核心读取
- `stations.json` 与 `routes.json` 供前端展示

### Python 是否替代了 C

不是。

C 负责核心路径搜索，Python 负责桥接与结果组织。

### 中文乱码是否一定意味着算法错误

不是。

中文乱码更常见的原因是：

- 文件编码不一致
- 终端显示编码不一致

不应把显示问题直接等同于算法错误。

## 11. 简要答辩表述

如果需要用简洁语言概括本项目，可以表述为：

“本项目将真实西安地铁网络抽象为图，节点采用‘站名 + 线路’状态节点，图使用邻接表存储，优先队列使用手写最小堆，路径搜索采用扩展 Dijkstra。系统再通过 Flask 连接 C 核心与前端页面，把查询结果以地图和乘车步骤的形式展示出来。”
